====== Tabul - How to list SQL Tables ======

===== About =====
This [[howto:howto|How-to]] will show you how to use the [[docs:tabul:data:list|tabul data list]] command to list <icon name="ant-design:table-outlined"/>  [[docs:resource:table|SQL Table]].


===== Steps =====



==== Create the TPCDS schema ====


  * Delete all tables to start with a clean state
<unit>
<code bash>
tabul data drop --not-strict *@sqlite
</code>
</unit>
  * Create all tables of the [[:docs:connection:tpcds|TPCDS schema]] to be able to demonstrate the query capabilities
with the [[:docs:tabul:data:create|create command]]
<unit>
<code bash>
tabul data create *@tpcds @sqlite
</code>
</unit>

==== Listing all tables ====

Listing all the tables from the [[docs:connection:tpcds|Tpcds connection]] with a [[docs:resource:data_selector|data uri selector]] and the star ''*'' [[docs:resource:data_selector#glob pattern|glob wildcard character]]

tabul data list *@sqlite
</code>

==== Listing one table with its dependencies ====

Listing the table ''store_sales'' from the [[docs:connection:tpcds|Tpcds connection]] and its [[docs:resource:dependency|dependencies]]
<unit>
<code bash>
tabul data list --with-dependencies store_sales@sqlite
</code>
<console>

</unit>

==== Filtering the output with a glob pattern ====

=== Listing all returns ===

Listing all the tables that have ''return'' in their name with a [[docs:resource:data_selector|data uri selector]] and its [[docs:resource:data_selector#glob pattern|glob pattern]]
<unit>
<code bash>
tabul data list *return*@sqlite
</code>
<console>
path
-----------------
catalog_returns
s_catalog_returns
s_store_returns
s_web_returns
store_returns
web_returns
</console>
</unit>

=== Listing all ''returns'' and ''sales'' tables ===

With [[docs:system:tpcds|tpcds]], the [[docs:resource:data_selector#glob pattern|glob expression]] ''{*returns,*sales}'' will returns all ''returns'' and ''sales'' tables.
<unit>
<code bash>
tabul data list {*returns,*sales}@sqlite
</code>
<console>
path
-----------------
catalog_returns
catalog_sales
s_catalog_returns
s_store_returns
s_web_returns
store_returns
store_sales
web_returns
web_sales
</console>
</unit>
